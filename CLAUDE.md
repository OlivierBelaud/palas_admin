## Batch Plan

### Batch A — Pino Logger (simplest, no external deps except pino)

**Package**: `packages/adapter-logger-pino/`

**Dependencies**: `pino`, `pino-pretty`

**What to implement**:
- `PinoLoggerAdapter` implements `ILoggerPort`
- Constructor takes `{ level?: string, pretty?: boolean }`
- `pretty: true` → uses `pino-pretty` transport (dev). `false` → JSON to stdout (prod)
- All 8 levels mapped: error→error, warn→warn, info→info, http→info, verbose→debug, debug→debug, silly→trace, panic→fatal
- `activity(msg)` → logs info with unique ID, returns ID
- `progress/success/failure(id, msg)` → logs with the activity ID
- `shouldLog(level)` → checks against Pino's level threshold
- `setLogLevel(level)` → `pino.level = level`
- `dispose()` → `pino.flush()` (no-op for stdout)

**Verify**: create `tests/adapters/logger-pino.test.ts` that instantiates PinoLoggerAdapter and runs LG-01→LG-08.

### Batch B — Drizzle + PG (the big one)

**Package**: `packages/adapter-drizzle-pg/`

**Dependencies**: `drizzle-orm`, `postgres`, `drizzle-kit`

#### B.1 — DrizzlePgAdapter implements IDatabasePort (SPEC-056)

- `initialize(config)` → create connection pool via `postgres` driver, wrap with `drizzle()`
- `dispose()` → `pool.end()` (~10ms per SPEC-071)
- `healthCheck()` → `SELECT 1` with 2s timeout
- `getClient()` → return drizzle instance
- `transaction(fn, options?)` → `db.transaction(fn)` with isolation level support
- Nested transactions → savepoints via `tx.transaction()` when `enableNestedTransactions: true`

#### B.2 — DrizzleRepository implements IRepository (SPEC-126)

- `find(options?)` → build SELECT with where, order, limit/offset, cursor pagination
- Auto-filter `WHERE deleted_at IS NULL` unless `withDeleted: true`
- `create(data)` → INSERT RETURNING with prefixed ID generation
- `update(data)` → UPDATE RETURNING, throw NOT_FOUND if 0 rows
- `delete(ids)` → DELETE WHERE id IN
- `softDelete(ids)` → UPDATE SET deleted_at = NOW(), return cascaded link IDs
- `restore(ids)` → UPDATE SET deleted_at = NULL
- `upsertWithReplace(data, replaceFields?, conflictTarget?)` → INSERT ON CONFLICT DO UPDATE
- `transaction(task, options?)` → delegate to db.transaction or reuse existing

#### B.3 — Error Mapper (SPEC-133)

```
PG 23505 (UNIQUE) → MantaError('DUPLICATE_ERROR')
PG 23503 (FK) → MantaError('NOT_FOUND')
PG 23502 (NOT NULL) → MantaError('INVALID_DATA')
PG 40001 (SERIALIZATION) → MantaError('CONFLICT')
Everything else → MantaError('DB_ERROR')
```

- Creates a test schema
- Instantiates DrizzlePgAdapter + DrizzleRepository

### Batch C — Nitro HTTP Adapter

**Package**: `packages/adapter-nitro/`

**Dependencies**: `h3`

#### C.1 — NitroAdapter implements IHttpPort (SPEC-039)

The 12-step pipeline wired to H3:

1. **RequestID** — generate/propagate `x-request-id`
2. **CORS** — set headers based on namespace (`/admin`, `/store`, `/auth`)
3. **RateLimit** — no-op pass-through (rate limiting disabled by default)
4. **Scope** — `container.createScope()` + `asyncLocalStorage.run(scope, ...)`
5. **BodyParser** — `readBody(event)` based on content-type
6. **Auth** — extract credentials from headers/cookies, call `IAuthGateway.authenticate()`
7. **PublishableKey** — validate `x-publishable-api-key` for `/store` routes
8. **Validation** — Zod schemas from `defineMiddlewares()` (no-op if none)
9. **Custom middlewares** — from `defineMiddlewares()`
10. **RBAC** — no-op if flag off
11. **Handler** — call the route handler
12. **ErrorHandler** — catch MantaError → HTTP status (SPEC-041)

#### C.2 — createMantaHandler()

```typescript
export function createMantaHandler(container: IContainer) {
  return defineEventHandler(async (event) => {
    const adapter = container.resolve<NitroAdapter>('IHttpPort')
    await adapter.handleRequest(event)
  })
}
```

#### C.3 — Health endpoints (SPEC-072)

- `/health/live` → 200 `{ status: "alive", uptime_ms }`
- `/health/ready` → 200/503 with checks on DB, cache, migrations

#### C.4 — Error mapping (SPEC-041)

```
NOT_FOUND → 404
INVALID_DATA → 400
UNAUTHORIZED → 401
FORBIDDEN → 403
DUPLICATE_ERROR → 422
CONFLICT → 409
NOT_ALLOWED → 400
DB_ERROR → 500
UNEXPECTED_STATE → 500
INVALID_STATE → 500
```

Body format: `{ type, message, code?, details? }`. No stack in prod.

export async function GET(req) {
  const productService = req.scope.resolve('productService')
  const products = await productService.listProducts(req.context)
  return Response.json({ products })

export async function POST(req) {
  const productService = req.scope.resolve('productService')
  const product = await productService.createProducts(req.context, req.validatedBody)
  return Response.json({ product }, { status: 201 })
}

## Rules

1. Each adapter is a **separate npm package** in `packages/`
2. Each adapter imports port interfaces from `@manta/core/ports`
3. Each adapter exports a class that `implements` exactly one port
4. No `any` types
5. Every file starts with `// SPEC-XXX — description`
6. Test each adapter against conformance suite after building
7. Log progress in `IMPLEMENTATION_LOG.md`
8. The demo app is the final proof — if it serves HTTP and talks to PG, it works
9. **Existing 314 conformance tests must still pass** after adding adapters
